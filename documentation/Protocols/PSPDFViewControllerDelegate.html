PSPDFViewControllerDelegate Protocol Reference

Conforms to NSObject
Declared in PSPDFViewControllerDelegate.h

Overview

register to this delegate in PSPDFViewController Note: page is based of 1 till pageCount

Tasks

  • – pdfViewController:willDisplayDocument:

    time to adjust PSPDFViewController before a PSPDFDocument is displayed
  • – pdfViewController:didDisplayDocument:

    delegate to be notified when pdfController finished loading
  • – pdfViewController:willShowPage:

    deprecated. called same time as didShowPage
  • – pdfViewController:didShowPage:

    controller did show/scrolled to a new page (at least 51% of it is visible)
  • – pdfViewController:didRenderPage:pdfScrollView:

    page was fully rendered at zoomlevel = 1
  • – pdfViewController:didChangeViewMode:

    will be called when viewMode changes
  • – pdfViewController:didTapOnPage:info:coordinates:

    if user tapped within page bounds, this will notify you. return YES if this touch was processed by you and need no further checking by PSPDFKit. Note that PSPDFPageInfo may has only page=1 if the optimization isAspectRatioEqual is enabled.
  • – pdfViewController:didTapOnAnnotation:page:info:coordinates:

    delegate for tapping on an annotation. If you don’t implement this or return false, it will be processed by default action (scroll to page, ask to open Safari)
  • – pdfViewController:willLoadPage:pdfScrollView:

    called before a pdf page will be loaded and added to the pagingScrollView
  • – pdfViewController:didLoadPage:pdfScrollView:

    called after pdf page has been loaded and added to the pagingScrollView
  • – pdfViewController:willUnloadPage:pdfScrollView:

    called before a pdf page will be unloaded and removed from the pagingScrollView
  • – pdfViewController:didUnloadPage:pdfScrollView:

    called after pdf page has been unloaded and removed from the pagingScrollView
